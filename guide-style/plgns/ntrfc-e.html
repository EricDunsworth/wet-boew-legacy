Interface enhancement

WET plug-ins, widgets, and styles should be used according to the purpose and audience of the Web page and should contribute to, or enhance, the usability of the page and the users' experience. They should be used for their intended purposes, and never to compensate for bad page design or poor document or content structure.

Calendar interface (includes date-picker)

Overview

This feature dynamically generates a calendar interface for navigating a list of events or for selecting a date in a form.

Examples:

  • Date picker - 2-column
  • Sélecteur de date - 2 colonnes
  • Events calendar - 2-column
  • Événements - 2 colonnes

Points of compliance

Use this feature only for its intended purposes, e.g. the date picker should only be used in conjunction with a form element requiring date input and the events calendar for navigating a chronological list of events.

For more information, refer to the Calendar interface - Feature guide.

Coda slider

Overview

This feature shows content to the user one section at a time. A small table of contents can be used to control which section of content is currently shown. A scrolling/sliding effect is used when changing the section content that is shown.

Examples:

  • Basic slider - horizontal
  • Coulisse simple - horizontale
  • Basic slider - vertical
  • Slider with grid - 2 x 4
  • Slider with grid - 3 x 3

Points of compliance

Use this feature only for its intended purposes to ensure a consistent user experience.

Recommended usage includes slideshow presentations.

For more information, refer to the Coda slider - Feature guide.

Expandable/collapsible content

Overview

This feature enables users to dynamically show or hide content within a Web page. Options include "Expand All" and "Hide All" buttons.

Examples:

  • Expandable/collapsible content
  • Contenu en arborescence affichable/masquable

Points of compliance

Use this feature only for its intended purposes to ensure a consistent user experience. Always follow these basics:

  • Use to hide secondary or tertiary content, as it is likely users rarely need to expand this information.
  • Use to hide content that users only need to read once.

Do not do the following:

  • Use to hide primary or essential content.
  • Use to tidy up the look of your page. If the page is deemed too busy, consider other options to arrange the content in a more usable way.

For more information, refer to the Expandable/collapsible content - Feature guide.

Geomap

Overview

This feature adds a Web map to your page through a simple wrapper for OpenLayers (http://www.openlayers.org). Anything that is possible with OpenLayers, is also possible with the Geomap feature. The Geomap plugin adds accessibility features on top of OpenLayers. This is a work in progress.

Examples:

  • Geomap - 1-column
  • Géocarte - 1-colonne
  • Geomap - 2-column
  • Géocarte - 2-colonnes

Points of compliance

Use this feature only for its intended purposes to ensure a consistent user experience.

Recommended usage includes diplaying information on a map or providing a map-based interface.

For more information, refer to the Geomap - Feature guide.

Multimedia player

Overview

This feature provides a multimedia player for playing videos on a Web page.

Examples:

  • Multimedia player (requires Flash 10.1/Silverlight 3 or higher)
  • Lecteur multimédia (nécessite Flash 10.1/Silverlight 3 ou plus haut)

Points of compliance

Use this feature only for its intended purposes to ensure a consistent user experience.

To meet accessibility guidelines, closed captions or descriptive audio must be provided.

For more information, refer to the Multimedia player - Feature guide.
