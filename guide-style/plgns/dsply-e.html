Display enhancement features

WET plug-ins, widgets, and styles should be used according to the purpose and audience of the Web page and should contribute to, or enhance, the usability of the page and the users' experience. They should be used for their intended purposes, and never to compensate for bad page design or poor document or content structure.

Archived Web Page template

Overview

This feature provides an "archive" notice for your archived content. The notice appears at the top of the browser window when the user scrolls away from the top of the page.

Examples:

  • Archived Web Page template (eng)
  • Modèle de page Web archivée (fra)

Points of compliance

Use this plug-in only for its intended purpose. This ensures a consistent user experience. It is intended for web pages that are:

  • maintained for reference, research or recordkeeping purposes;
  • will not be altered or updated after the date of archiving; and
  • stored in a digital repository.

For more information, refer to Archived Web Page template - Feature guide.

Charts and graphs support

Overview

This feature dynamically generates charts and graphs from table data. The table data used to generate the graphic need to have a structure as defined in the Tables documentation. The graphic generated can be customized by using the CSS Option Class.

Examples:

  • Basic charts and graphs - 1-col page
  • Basic charts and graphs - 2-col page
  • Complex charts and graphs - 1-col page
  • Complex charts and graphs - 2-col page

Points of compliance

Use this feature only for its intended purpose. This ensures a consistent user experience. For correct use of this plug-in, it is recommended that you follow these basics:

  • Summarize the chart values in a paragraph above the chart.
  • Place tables and their corresponding graphs in close proximity to each other. The table must appear first, then the graph.
  • Provide a "skip" link before each graph or section of graphs, and a corresponding anchor after each graph or section. This allows adaptive technology users to by-pass graphs they cannot access and allows them to skip to the tables.

For more information, refer to Charts and graphs support - Feature guide.

Progress bar

Overview

The purpose of this feature is to provide an accessible progress bar widget that can be used to:

  • display a static progress to the user, or
  • display the progress of an asynchronous task.

Examples:

  • Progress bar (eng)
  • Barre de progression (fra)

Points of compliance

Use this feature only for its intended purpose. This ensures a consistent user experience.

For more information, refer to Progress bar - Feature guide.

Search term highlighting

Overview

This feature automatically highlights certain words on a Web page. The words to be highlighted can be set manually or through a simple search form.

Note: This feature was designed to highlight words for visual users and may not be of much use to users with a visual impairment.

Examples:

  • Search term highlighting (eng)
  • Mise en surbrillance des termes recherchés (fra)

Points of compliance

Use this feature only for its intended purpose. The recommended usage includes:

  • highlighting search terms on a search results page,
  • highlighting key words on a Web page, or
  • displaying all instances of one or more words on a Web page.

For more information, refer to Search term highlighting - Feature guide.

Tracked changes

Overview

This feature visually identifies tracked changes and provides a mechanism for navigating tracked changes in HTML documents.

Examples:

  • Tracked changes (eng)
  • Suivi des modifications (fra)

Points of compliance

Use this feature only for its intended purpose. This ensures a consistent user experience.

For more information, refer to Tracked changes - Feature guide.

Zebra striping

Overview

This feature adds zebra stripes to lists and tables by colouring alternate list items and table rows.

Examples:

  • Zebra striping (eng)
  • Rayage du zèbre (fra)

Points of compliance

Use this feature only for its intended purpose. The recommended usage includes:

  • data tables (especially complex/long tables)
  • complex lists (such as user feedback/comments messages)
  • forms

For more information, refer to Zebra striping - Feature guide.
