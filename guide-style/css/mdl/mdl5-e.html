Overview of module-table-contents

Introducing CSS class:

module-table-contents

Use the module-table-contents class for creating a table of contents. This class works with a span-4, which can have one or two columns of links, or a span-6, which can have one, two or three columns of links.

In general, modules are specific CSS classes that work within the grid system to create an entire presentation style. While all module CSS are found here, this particular module creates the following effect:

Points of compliance

Only use this module for its intended purpose. This ensures that any changes made to the CSS have predictable outcomes, and that there is consistent user experience.

Correct use of module-table-contents

When using this module, always follow these basics:

  • Ensure this module appears on the left side of the Web page.
  • Place at the top of any page that scrolls when viewed at 800x600 screen resolution (unless scrolling is minimal, and adding a Table of contents is no value to the user).
  • List only links within a table of contents. A table of contents usually links to anchors on the same page. In the case of an extremely large table of contents, it can take users to anchors on another page.
    • For large documents, break the main table of contents into appropriately titled menu sections. Do this is to avoid using the title "Table of contents" for both a main and a submenu page of a large document. It's not about where the hyperlinks go. It's about avoiding the redundant use of the "Table of contents" title.
  • Use within a grid system, but only within a span-4 or a span-6. For example: <div class="span-4 module-table-contents"> or a <div class="span-6 border-span-6 module-table-contents">.
  • Use a span-4 table of contents — only if it doesn't crowd the content. By doing so, you can create a pocket of white space to the right of the table of contents (an empty span-2). White space creates breathing room and balance on the Web page. So if there is no need, don't fill this span-2 with a right menu module.
  • Manually include a border for a span-6 for example, <div class="span-4 module-table-contents"> versus <div class="span-6 border-span-6 module-table-contents">. The border automatically appears on a span-4, but cascades to a span-6. Therefore, for the border to appear properly in a span-6, you must override the default border-span-4 with the border-span-6 style.
  • Include a maximum of two or three levels within a table of contents list. Any more than three levels tends to confuse the user, and it is more difficult to maintain.
  • Begin each listing with a <p> tag titled "Table of contents" for example, <p>Table of contents</p>. This is at the request of Adaptive Technology.
  • Do not capitalize the word "contents". Refer to The Canadian Style sections 4.31 Headings, 11.06 Table of contents, and 11.6 Headings.

Incorrect use of module-table-contents

Do not do the following:

  • Use this module in a manner that contradicts its intend use.
  • Use this module on a Welcome page. Instead, use module-welcomepage
