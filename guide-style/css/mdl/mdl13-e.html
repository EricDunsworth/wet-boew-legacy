Overview of module-news and module-spotlight

Introducing CSS class:

  • module-news
  • module-spotlight
  • module-news-article

The module-news and module-spotlight classes are for news articles and stories. The module-news class allows for an embedded article or story and it works in conjunction with the module-news-article class. The module-spotlight showcases a list of links to articles or stories. module-news works with span-2 and span-3, while module-spotlight only works with span-3.

In general, modules are specific CSS classes that work within the grid system to create an entire presentation style. While all module CSS are found here, this particular module creates the following effect:

Source code and appearance options

The module-news and module-spotlight have several layout options. Below each example is the corresponding source code that you can select and copy.

Note: module-news-article adds a border around the content. So if you want to have several columns of equal height, you need to apply the borderX style to the parent grid. However, this generates two borders, and is not recommended. Manually adjust the amount of text in each grid cell if you want side-by-side articles the same height.

Points of compliance

Only use these modules for their intended purpose. This ensures that any changes made to the CSS have predictable outcomes, and that there is consistent user experience.

Correct use of module-news and module-spotlight

When using these modules, always follow these basics:

  • Use module-news to embed articles or stories.
  • Use only within a grid system (span-2 to span-6). For example: <div class="span-3 module-news">.
  • Use module-spotlight to link to articles or stories. The module-news works within a span-3 for example, <div class="span-3 module-spotlight">.
  • Use both of the modules for informal content such as news stories and articles that are found in newsletters or event notices. The flexibility of the grid system allows you to place these modules anywhere within the system (they do not need to left-align).
  • Start each module with a heading (<h2> to <h6>). Customize the heading text to suit the nature of the links.
  • Make the article heading a link if the article includes a "Continue reading" link.
  • Ensure links are descriptive for users with assistive technologies. For all generic links like "Continue reading" or "(x) comments", wrap the name of the article in the cn-invisible class. For example: <p><a href="#">Continue reading<span class="cn-invisible">: A decade of excellence</span></a> <span><a href="#">4 Comments<span class="cn-invisible"> made on the article: A decade of excellence</span></a></span></p>
  • Manually adjust the amount of text in each grid cell if you want side-by-side articles the same height. The equalize CSS does not achieve symmetry. It is the text inside the grid cell that has the border, not the grid cell itself. If you apply the equalize CSS, it stretches the grid cell to match the adjacent grid cell. However, there is no visual change, as this style does not affect the article inside the grid.

Incorrect use of module-news and module-spotlight

Do not do the following:

  • Use this module in a manner that contradicts its intend use.
  • Use these modules for formal work-related tax content.
  • Adjust the default background colours.
