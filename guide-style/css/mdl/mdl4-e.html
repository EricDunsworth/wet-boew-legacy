Overview of module-event

Introducing CSS class:

module-event

An events menu must include a link to the main events page and can optionally include a calendar. This type of menu uses the class module-event and works within a span-2.

In general, modules are specific CSS classes that work within the grid system to create an entire presentation style. While all module CSS are found here, this particular module creates the following effect:

Floating technique

There are times when you want to place the module on the right side of a page, and have the neighbouring content wrap around and below it. To achieve this, apply a float-right to the module, but not in a grid cell. Then place the content after the module.The content then freely wraps around the module and fills all the available white space.

Points of compliance

Use this module only for its intended purpose. This ensures predictable outcomes to any changes made to the CSS, and that the user experience is consistent.

Correct use of module-event

When using this module, always follow these basics:

  • Place on the right side of the Web page with a link to an events page. You have the option to add a table calendar.
  • Title the link to the events page "View all events in detail".
  • Create your own local solution, or manually maintain the event details. This template does not provide the application that dynamically adds the calendar to your page. Therefore, if you add a calendar table to the module-event, it must do the following:
    • Appear below the link to the events page.
    • Use a single character format for the days of the week in order to fit in the available space. The days of the week use acronyms for example <abbr title="Monday">M</abbr>.
    • Use descriptive calendar links. For example, <a href="#">13</a> does not meet CLF requirements, as the link is not descriptive. Therefore, all calendar date links must include the date details, wrapped in the cn-invisible class. For example: <a href="#">13<span class="cn-invisible">th of October events</span></a>. Ensure there is no spacing between a number and the span text "st", for example. Otherwise, when you use the invisible class, JAWS reads the number 1 and "st" as two words, not as the word "first".
    • Use the calendar-today class for the current day <td> cell for example, <td class="calendar-today">9</td>
    • Use the calendar-month class for the current month in the <table> for example, <table class="calendar-month">.
    • Code the table caption with calendar month and year for example, <caption>October 2009</caption>
    • Use the calendar-weekend class for all weekend days in the header and table rows.
  • Use within a grid system, but only within a span-2. For example: <div class="span-2 module-event">.
  • Use on any Web page, as needed.
  • Start each list with a heading ( <h2> to <h6> all work). Customize the heading text, as appropriate. Otherwise use the title "Events".
  • Include the class span-footer for example, <div class="span-2 module-event span-footer">, unless you stack this module with other modules. Then only the final module has the span-footer class.
  • Place stacked modules (children) inside a parent span-2 . This parent contains these grid modules so they don't float to other spots on the page. Anytime you establish a grid parent and grid child relationship, you normally have to apply the row-end class to each child module. However, the CSS in this template forecasts this specific relationship and accounts for it. So in this case, there is no need to apply the row-end class. If you apply the row-end class out of habit, there is no harm.

Incorrect use of module-event

Do not do the following:

Use this module in a manner that contradicts its intend use.
