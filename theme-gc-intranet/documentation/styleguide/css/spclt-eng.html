Invisible content CSS

Introducing CSS class:

cn-invisible

The cn-invisible class clips content into a one pixel square. It allows you to create links that are descriptive (and compliant), but not visually clutter the page. It is common to use this technique to target assistive technology users. For example, JAWS users hear the information, but the visual user does not hear or see it. The CSS attribute display:none; does not achieve the same effect, as JAWS ignores this style entirely.

The solution of moving content out of view gives webmasters the ability to create imbedded messages, and enrich link text that is fully descriptive.

It is common to use cn-invisible in the following situations:

  • News and spotlight CSS modules — module-news and module-spotlight
  • Embedded navigation CSS

Source code


<p><a href="#">Continue reading<span class="cn-invisible">: (Name of article)</span></a></p>

Points of compliance

The following points always apply:

  • Include a <span class="cn-invisible"> with all generic and non-descriptive links. Use this class to visually hide the text, and to turn a generic link into a descriptive link. Refer to W3C, C7: Using CSS to hide a portion of the link text (external link).
  • Ensure you have proper spacing between the start and end of the visible and invisible text. Otherwise, two words can read as one to the JAWS user. Put spacing inside the <span> tag when leading with cn-invisible.

"New" CSS

Introducing CSS class:

new

Use this class to draw attention to new content. Mainly use it to showcase new hyperlinks that you add to a list of pre-existing links. For example:

  • Fake link
  • Fake link New
  • Fake link

Source code


<ul>
  <li><a href="#">Fake link</a></li>
  <li><a href="#">Fake link</a> <span class="new">New</span></li>
  <li><a href="#">Fake link</a></li>
</ul>

Points of compliance

The following point always applies:

  • Remove this class from your content after 30 days, as the content is no longer new.

"Update" CSS

Introducing CSS class:

update

Use this class to draw attention to new content. Mainly use it to showcase new hyperlinks that you add to a list of pre-existing links. For example:

  • Fake link
  • Fake link Updated - February 6, 2012
  • Fake link

Source code


<ul>
  <li><a href="#">Fake link</a></li>
  <li><a href="#">Fake link</a> <span class="update">Updated - February 6, 2012</span></li>
  <li><a href="#">Fake link</a></li>

The following point always applies:

  • Adding a date adds value to the user but is not required.

No wrap CSS

Introducing CSS class:

wrap-none

This class prevents content from splitting over multiple lines. Use it mainly when writing something such as a date, phone number or mathematical equation, as this ensures they are easy to read.

Without the wrap-none class applied to the date:

With the wrap-none class applied to the date:

Fake text fake text fake text fake text fake text fake text fake text fake text fake text fake text fake text fake text fake March 1, 2010

Source code


<span class="wrap-none">January 1, 2010</span>

Points of compliance

Use when appropriate — remember to test your Web page in both 800x600 and 1024x768 resolutions.

Thermometer CSS

Introducing CSS classes:

  • thermometer-0
  • thermometer-10
  • thermometer-20
  • thermometer-30
  • thermometer-40
  • thermometer-50
  • thermometer-60
  • thermometer-70
  • thermometer-80
  • thermometer-90
  • thermometer-100
  • thermometer-110
  • thermometer-120

Use this style is to record the donation progress for events such as the Government of Canada Workplace Charitable Campaign (GCWCC).

This CSS determines which thermometer background image to display. By changing the class name on the <p> element, the thermometer image instantly reflects the new amount. Since the classes increase by 10% increments, you need to adjust your thermometer percentage to the nearest 10% class. Values range from 0% to 120%. For values in excess of 120% , use the 120% class.

Donations to date are $0, which is 0% of the target.

(Uses thermometer-0)

Donations to date are $46,500, which is 38% of the target.

(Uses thermometer-40)

Donations to date are $190,512, which is 150% of the target.

(Uses thermometer-120)

Source code

<div class="span-2">
<p class="thermometer-0">Donations to date are $0, which is 0% of the target.</p>
</div>
<div class="span-2 border-span-2">
<p class="thermometer-40">Donations to date are $46,500, which is 38% of the target.</p>
</div>
<div class="span-2 border-span-2 background-light">
<p class="thermometer-120">Donations to date are $190,512, which is 150% of the target.</p>
</div>

Points of compliance

The following points always apply:

  • Provide all the details in the <p> tag, as accessibility requirements state you cannot convey meaning with an image.
  • Take the actual percentage and round it up or down to the nearest 5% value, and then use the appropriate CSS class name. For percentages in excess of 120%, apply the thermometerMercury120 to the image, and describe the actual percent in the <p> tag that is above the image.
  • Use with GCWCC campaigns

The following points always apply:

  • Include the week column and the weekend columns if you need to — they are optional. If used, include the class: calendar-weekend in the header and table rows of all weekend days. You do not need a class for the "week" column.

Footnote CSS

Introducing CSS classes:

  • footnote
  • footnote-container

A footnote is a note of reference (a citation) or a comment. It provides details about something on that page; it is something related to, but of lesser importance than the main text. Place footnotes at the bottom of the page.

Example: Footnote and details (footnote-container) it links to on the page

Fake information fake information fake information fake information. Footnote 1 Fake information fake information fake information fake information.

Fake information fake information fake information fake information. Footnote 2

Footnote 1 Footnote (footnote-container) details go here. (Return to footnote 1 source paragraph)

Footnote 2 Footnote details go here. (Return to footnote 2 source paragraph)

Source code


<p>Fake information. <a id="fn1_source"></a><a href="#fn1" class="footnote">Footnote 1</a></p>
<p>Fake information. <a id="fn2_source"></a><a href="#fn2" class="footnote">Footnote 2</a></p>
<div class="footnote-container">
  <p><a id="fn1"></a>Footnote 1 <span class="footnote">Footnote details  go here. <a href="#fn1_source">(Return to footnote 1 source paragraph)</a></span></p>
  <p><a id="fn2"></a>Footnote 2 <span class="footnote">Footnote details  go here. <a href="#fn2_source">(Return to footnote 2 source paragraph)</a></span></p>
</div>

Points of compliance

The following points always apply:

  • Use the following format for all links to the footnote: 
    <a id="fn1_source" name="fn1_source"></a> <a href="#fn1" class="footnote">Footnote 1</a>
  • Use the following format for all links from the footnote: 
    <a id="fn1" name="fn1"></a><p>Footnote 1 <span class="footnote">Footnote details  go here. <a href="#fn1_source">(Return to footnote 1 source paragraph)</a></span></p>
  • Use only unique and descriptive links. You can use the words footnote and endnote interchangeably — just use them consistently. Refer to The Canadian Style (search for Reference notes).

Points of compliance

The following points always apply:

  • Align left at all times.
  • Must be inside a paragraph <p>
  • Include the cn-invisible class, as well as extra text to make links descriptive for visually impaired users. This is a requirement for all single character and non-descriptive links.
  • Ensure there is no spacing between a number and the span text "st", for example. Otherwise, when you use the cn-invisible class, JAWS reads the number 1 and "st" as two words, not as the word "first".
    • The correct way to code without spacing is as follows: 
      <a href="#">1<span class="cn-invisible">st page of search results</span></a>
    • The incorrect way to code with spacing is as follows: 
      <a href="#">1 <span class="cn-invisible">st page of search results</span></a>
<a href="#">1<span class="cn-invisible"> st page of search results</span></a>
  • Ensure there is spacing between the visible word and the span text "page of search results", for example. Otherwise, when you use the cn-invisible class, JAWS reads the text "Next" and "page" as a nonsense word.
    • The correct way to code with spacing is as follows: 
      <a href="#">Next<span class="cn-invisible"> page of search results</span></a>
    • The incorrect way to code without spacing is as follows: 
      <a href="#">Next<span class="cn-invisible">page of search results</span></a>
  • Use the word “to” rather than a dash in a heading for example, A to Z, not A–Z. Otherwise, the heading is not descriptive, nor clear for the visually impaired JAWS user.
  • Ensure the navigational links appear as a single row and don't wrap — unless there is a valid reason for this to occur.
  • Create the link text that suits the nature of the local embedded navigation. For example, use the text "Previous article, Next article" or "Previous page".
  • Choose the appropriate amount of navigational links. There is no preset number that works in all situations.

Teaser CSS

Introducing CSS classes:

  • teaser-1
  • teaser-2

Used to draw attention trigger words. This acts as a teaser to the information that immediately follows.

Class Details Example 1: With a heading Example 2: With a list Example 3: With a link
teaser-1 adds decoration to the trigger keywords

Step 1 Do x
  • Option 1 X option

Did you know? Text
teaser-2 adds decoration to the trigger keywords

Step 1 Do x
  • Option 1 X option

Did you know? Text

Points of compliance

The following points always apply:

  • Content following the teaser must directly relate the text inside the teaser.
  • Teaser can include a link.
  • Teaser can be used with headings, paragraphs and lists.
  • Teaser can only contain a few words. For example:

    • Step 3 Read the style guide

    • Live demo 2 column layout
