Overview of module-versatile

Introducing CSS class:

module-versatile

The module-versatile class allows for any type of content in a right menu. To simply feature a list of links that point to related Web pages, refer to Right menu: Related links CSS module.

This module works with a span-2. In general, modules are specific CSS classes that work within the grid system to create an entire presentation style. While all module CSS are found here, this particular module creates the following effect:

Floating technique

There are times when you want to place the module on the right side of a page, and have the neighbouring content wrap around and below it. To achieve this, apply a float-right to the module. Then place the content after the module, but not in a grid cell. The content then freely wraps around the module and fills all the available white space.

Points of compliance

Use this module only for its intended purpose. This ensures predictable outcomes to any changes made to the CSS, and that the user experience is consistent.

Correct use of module-versatile

When using this module, always follow these basics:

  • Feature any type of content.
  • Ensure this module appears on the right side of the Web page.
  • Use within a grid system, but only within a span-2. For example: <div class="span-2 module-versatile">
  • Use on any Web page, as needed. It can appear multiple times throughout the page.
  • Combine with a float-right , as needed.
  • Customize the heading name, as appropriate. A module-versatile does not need to start with a heading, as you can use it as a basic right menu feature box. However, if you use a heading, an <h2> to <h6> all work.
  • Put the image inside a <p> tag to add a 10 pixel padding (as in the first example above), or apply the image-actual class so the image retains its original dimensions. Refer to Image CSS and grids for full details. Images in a grid system automatically stretch to the full width of a grid cell, which means any images placed inside the module-versatile always stretch the image to 224 pixels or 194 pixels, depending on the user's resolution.
  • Place stacked modules (children) inside a parent span-2 . This parent contains these grid modules so they don't float to other spots on the page. Anytime you establish a grid parent and grid child relationship, you normally have to apply the row-end class to each child module. However, the CSS in this template forecasts this specific relationship and accounts for it. So in this case, there is no need to apply the row-end class. If you apply the row-end class out of habit, there is no harm.

Incorrect use of module-versatile

Do not do the following:

  • Use this module in a manner that contradicts its intend use.
  • Stack too many module-versatile items together. Rather, add another heading so you can stack related items together in smaller groupings.
  • Change the default background colour.
  • Apply the module Footer to the module-versatile Both the footer and the module-versatile border are background images. CSS only allows for one background image, not two.
