Background

When a link performs more than one action (on first click it expands content and the second click it collapses content), the link text must dynamically change to inform the user what state the information is in. While many sites use the +/- as link text, it is not descriptive enough for visually impaired users (plus text-to-speech software like JAWS does not read the "-" as a minus and instead states that it is a dash), plus the link target is typically quite small and can be difficult to click on for users that have dexterity difficulties.

One of the key reasons a link needs to be descriptive is because a visually impaired user can go into "links mode" and listen to all the links on the page, without additional context, so having 5 links that all say"expand" or "+" is not helpful to them and therefore does not pass accessibility requirements.

As of 2010, most of the mainstream Internet sites use imagery, like rotating arrows or plus/minus signs, to let users know when information is expanded or hidden. Therefore, this GoC solution will mirror the industry norm.

What this script will do

This jQuery solution not only mimics the industry norm of using associative imagery, but also allows you to create unique and descriptive link text to inform all users of the state of the content.

The solution works in both the left menu and in the core content area and dynamically toggles the expand/hide text between English and French, based on the meta language type.

How to use this script

  1. Add this line:
    expandhide: {titleSelector:".toggle-link-text", globalToggles:true}
    to the editable script area of your progressive enhancement script in the <head>, so it looks like this:
    <script type="text/javascript"> /* <![CDATA[ */ var params = { expandhide: {titleSelector:".toggle-link-text", globalToggles:true} }; PE.progress(params); /* ]]> */ </script>
  2. The titleSelector:".toggle-link-text" feature ensures that expand/hide links are customizable and descriptive and allows you the flexibility to change a undescriptive link like "Expand", into a descriptive link like "Expand: Example 1". The plugin extracts the text out of the element that has the toggle-link-text class applied to it, and injects that text into the expand/hide link. Therefore,add this class to the any element within the toggle-content container. For example the link would say "Expand: Example 2" and "Hide: Example 2":
    <div class="toggle-content collapse">
    <h2 class="toggle-link-text">Example 2</h2>
  3. The globalToggles:true can be true or false, where true means that you want "Expand all" and "Hide all" buttons on the page and false means you don't. The plugin will automatically place these buttons above the first expand/hide link. However, if you want to specify a different location, place this code <div class="toggle-all-controls"></div> where you want the buttons to appear and the plugin will place the buttons there instead. You can use this code several times on a page to create multiple occurrence of these buttons.
  4. You must wrap the content that is to be expanded and hidden with <div class="toggle-content">(all the content)</div>. If you want the content collapsed by default, include a class collapse like so: <div class="toggle-content collapse">(all the content)</div>.
